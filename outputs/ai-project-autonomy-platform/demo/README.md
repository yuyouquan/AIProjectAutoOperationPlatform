# Demo 1.0 演示说明

## 一句话介绍

AI 项目自治运营平台把项目管理、缺陷、体验大数据和售后/VOC 数据自动汇聚，按项目节点识别风险，生成任务建议，推动跟踪闭环，并自动输出周会摘要。

## 推荐演示路径

1. 打开本地 `http://localhost:4173/` 或线上 `https://ai-project-auto-operation-platform.vercel.app`，先讲首屏四个指标：接入信号、样例项目、识别风险、报告耗时。
2. 在“AI 自动巡检演示台”点击“开始 AI 自动巡检”，让评委看到 AI 自动读取项目管理、缺陷、大数据、售后/VOC 数据。
3. 观察每条巡检任务下方的解析证据：每一步都会直接展开原始数据解析、解析结果、AI 思考过程和决策输出，让评委看到 AI 为什么这么判断。
4. 观察执行日志逐步完成：完整巡检约 14 秒，适合边讲边看读取节点、读取缺陷、读取体验指标、读取 VOC、判断风险、生成任务、模拟写入、输出周会摘要。
5. 展示“AI 生成的任务草稿”，说明任务字段不是手工填的，而是由 AI 根据证据、责任和验收口径自动生成。
6. 向下看“AI 任务生成”和“自治跟踪看板”，证明模拟写入后任务表、看板和报告会同步变化。
7. 复制“周会报告输出”，说明这就是项目经理周会前最耗时的人工整理工作。
8. 最后看“参赛要求覆盖”，回答评委关于既有项目增量、业务价值和可落地性的关注。

## 现场讲法

“我们不是再做一个周报生成器，而是让 AI 替项目经理跑一遍巡检流程。Demo 1.0 展示的是一条自动闭环：AI 读取项目节点、缺陷遗留、体验大数据和售后/VOC 信号，先把原始数据解析成节点规则和风险证据，再展示 AI 为什么判断、为什么生成任务，最后生成带证据、责任人、截止时间和验收口径的任务建议，并同步到看板和周会报告。正式系统写回会作为下一步集成，当前 Demo 先证明业务逻辑和可落地路径。”

## 验收结论

- 桌面端已验证：无横向溢出、首屏文本无重叠、按钮尺寸达标。
- 移动端已验证：无横向溢出、首屏能看到下一段业务内容、按钮尺寸达标。
- 交互已验证：项目切换、任务模拟创建、看板同步、报告同步均可用。
