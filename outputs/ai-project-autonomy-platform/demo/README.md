# Demo 1.0 演示说明

## 一句话介绍

AI 项目任务智能体工作台把项目交付件、群聊、云文档和多系统原始数据自动解析成风险、任务、催办预警和飞书周会文档，让评委看到 AI 如何思考、如何分发任务、如何检查交付件质量。

## 推荐演示路径

1. 打开本地 `http://localhost:4173/` 或线上 `https://ai-project-auto-operation-platform.vercel.app`，先讲首屏四个指标：自动读取、AI 识别、任务生成、报告输出。
2. 在顶部“方案输入”选择项目场景，说明 Demo 读取的是交付件、群聊、云文档和 SPUG/TOnes、IPM、UTP、BOM、SAP、MES、MOM、QMS、CRM、Trancare 等系统信号。
3. 点击“开始 AI 分析”，中间栏会逐步展示原始数据、解析结果、AI 思考过程和决策输出，让评委看到 AI 为什么这样判断。
4. 观察右侧“任务闭环中枢”：AI 自动生成 4 条任务，字段包含背景说明、参考信息、清晰目标、责任人、截止时间、证据来源和验收口径。
5. 点击“交付件质检”的“一键上传”，模拟责任人上传交付件和进展说明；再点击“AI 建议打回”，展示智能体如何检测完成质量并给出补充建议。
6. 查看“催办与上升预警”，说明未完成和超期任务会提醒责任人，并预警任务下发人。
7. 查看“飞书文档输出”，说明周报、月报和上升预警最终会沉淀到飞书文档。
8. 最后看“参赛要求覆盖”，回答评委关于既有项目增量、业务价值和可落地性的关注。

## 现场讲法

“我们不是再做一个周报生成器，而是让 AI 替项目经理把收集、判断、建任务、催办和写报告串成一条可见流程。Demo 2.0 左侧展示方案输入和系统数据源，中间展示 AI 的原始数据解析、解析结果、思考依据和决策输出，右侧展示任务自动生成、一键分发、交付件质检、AI 建议打回、催办预警和飞书文档输出。正式系统写回会作为下一步集成，当前 Demo 先证明业务逻辑、交互体感和可落地路径。”

## 验收结论

- 桌面端已验证：无横向溢出、首屏文本无重叠、按钮尺寸达标，AI 思考卡片自然展开，无大块空白。
- 移动端已验证：无横向溢出，页面按方案输入、AI 思考、任务闭环、飞书输出纵向展开。
- 交互已验证：项目切换、AI 分析、任务生成、一键分发、交付件上传、AI 建议打回、催办预警和飞书报告输出均可用。
