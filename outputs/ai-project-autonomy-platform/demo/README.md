# Demo 3.0 演示说明

## 一句话介绍

AI 项目自治演示舱以真实飞书 Wiki `tOS交付进展汇报W23` 为样例，让评委看到 AI 如何把周会文档、项目管理系统、缺陷系统、售后系统和站点反馈里的风险，自动解析成任务、飞书通知、催办预警和可编辑的问题跟进列表。

## 推荐演示路径

1. 打开本地 `http://localhost:4173/demo/` 或线上 `https://ai-project-auto-operation-platform.vercel.app/demo/`。
2. 先讲左侧真实输入：tOS17.0 需求/Slim/IR 未锁定、整机构建一次成功率 47%、APR TOP10 解决率 1% (30/330)、Note 60 Pro 新增负向舆情 67 条、站点 Open 49/超期19。
3. 点击唯一按钮“开始 AI 自动演示”，让 Demo 自动跑完 `数据解析 -> AI 思考 -> 任务生成 -> 问题输出`，同时观察问题输出区的扫描表格等待特效。
4. 到“AI 思考”时重点讲：这不是周报排版，而是把构建、质量、VOC、站点反馈关联起来，判断哪些风险需要变成管理任务。
5. 到“任务生成”时重点讲：任务会带背景说明、参考信息、目标、责任人、截止时间和验收口径，并向于佑全、涂良建等责任人发送飞书通知。
6. 到“问题输出”时重点讲：最终沉淀为类似 IPM 的问题跟进列表，责任人/部门带头像和姓名，处理进展、状态、完成时间可以继续修改，后续可做人审后写回系统。

## 现场讲法

“我们这次不是做一个更漂亮的周报工具，而是让 AI 替项目经理读 W23 周会文档和各系统数据。它会先找出真实高风险信号，再解释为什么这些风险会影响挂测、上市和管理决策，最后生成可分发任务、通知责任人，并沉淀为可维护的问题跟进列表。老板能一眼看到：哪个版本风险最高、哪个责任人要在什么日期闭环、哪些问题需要上升预警。”

## 验收结论

- 桌面端已验证：一屏四步、唯一主操作、无横向溢出、玻璃态样式加载正常；输入区和 AI 区已调整为接近 50/50，整体纵向压缩，完成后问题表顶部进入首屏。
- 移动端已验证：内容纵向展开，按钮尺寸达标，文本无重叠。
- 业务数据已验证：Demo 包含 `tOS交付进展汇报W23`、`tOS17.0`、`47%`、`1% (30/330)`、`Note 60 Pro`、`Open 49` 和 `超期19` 等真实样例信号，并展示责任人头像、飞书通知与可编辑问题字段。
