# Demo 1.0 演示说明

## 一句话介绍

AI 项目自治运营平台把项目管理、缺陷、体验大数据和售后/VOC 数据自动汇聚，按项目节点识别风险，生成任务建议，推动跟踪闭环，并自动输出周会摘要。

## 推荐演示路径

1. 打开本地 `http://localhost:4173/` 或线上 `https://ai-project-auto-operation-platform.vercel.app`，先讲首屏四个指标：接入信号、样例项目、识别风险、报告耗时。
2. 看“数据源总览”，说明 Demo 使用四类真实业务系统字段做样例，不依赖生产写回。
3. 看“AI 推理链路”，强调 AI 不是只写报告，而是先判断节点、再识别风险、再生成任务。
4. 在“节点风险雷达”切换 `tOS17.0` 或 `TECNO CAMON 60 Pro 5G`，展示不同节点的风险判断。
5. 在“AI 任务生成”点击“模拟创建任务”，观察任务按钮、看板列和报告文本同步变化。
6. 复制“周会报告输出”，说明这就是项目经理周会前最耗时的人工整理工作。
7. 最后看“参赛要求覆盖”，回答评委关于既有项目增量、业务价值和可落地性的关注。

## 现场讲法

“我们不是再做一个周报生成器，而是把项目运行中的风险信号变成可追踪任务。Demo 1.0 证明了一条闭环：多系统信号进入后，AI 按项目节点判断风险，生成带证据、责任人、截止时间和验收口径的任务建议，再同步到看板和周会报告。正式系统写回会作为下一步集成，当前 Demo 先证明业务逻辑和可落地路径。”

## 验收结论

- 桌面端已验证：无横向溢出、首屏文本无重叠、按钮尺寸达标。
- 移动端已验证：无横向溢出、首屏能看到下一段业务内容、按钮尺寸达标。
- 交互已验证：项目切换、任务模拟创建、看板同步、报告同步均可用。
