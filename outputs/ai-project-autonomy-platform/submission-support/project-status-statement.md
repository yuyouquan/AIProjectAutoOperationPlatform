# 立项情况说明

项目名称：AI 项目自治运营平台

## 是否为既有立项项目

本项目基于团队对现有项目管理、周会、问题跟进和项目状态数据的理解开展，不把既有系统成熟度作为评审优势。本次参赛材料会客观说明已有基础，并重点展示赛事期间新增的 AI 能力、Demo 3.0 和真实 W23 业务闭环验证。

## 已有基础

- IPM 项目周会管理中已有会议轮值、项目概况、问题跟进、部门汇报、制度与模板等数据基础。
- 周会 Wiki/表格中已有项目状态、风险任务、责任部门、责任人、计划完成时间、状态等字段。
- 团队熟悉项目管理过程中的报告整理、风险跟踪、任务催办和周会汇报痛点。

## 本次赛事新增增量

- 新增 AI 节点风险推理：根据 KO / PR / QR / STR / PIR / MPR / MQR 等节点切换风险检查重点。
- 新增风险到任务生成：把风险转成带证据、责任部门、责任人、截止时间和验收口径的任务建议。
- 新增交互 Demo 3.0：以 `tOS交付进展汇报W23` 为样例，实现数据解析、AI 思考、任务生成、催办上升预警和飞书周会报告输出。
- 新增验证数据与讲解材料：用 CAMON / tOS 样例数据证明闭环可演示、可解释、可继续集成。

## 可落地路径

1. 先读后写：优先读取项目管理、缺陷、大数据、售后系统的关键字段，降低改造风险。
2. 人审后写回：Demo 3.0 先生成任务建议，正式写回任务系统作为下一步集成。
3. 按节点逐步扩展：先覆盖高频周会和高风险项目，再扩展到更多产品线和节点。
4. 形成闭环指标：跟踪风险发现提前量、报告整理耗时、任务关闭率、逾期升级率等指标。
